# Machine Learning Architecture in Byte

Byte is not just a hardcoded state machine; it is an intelligent, autonomous entity. This document details the hybrid Machine Learning (ML) approach used to achieve a dynamic, hyper-personalized, and privacy-first AI companion.

## 1. The Hybrid Intelligence Approach

Byte achieves intelligence by separating concerns across three distinct types of ML models, all running **locally** on your macOS device:

*   **Reinforcement Learning (Q-Learning):** Drives autonomous physical actions and routines without requiring a GPU.
*   **Large Language Models (LLM):** Powers conversational intent, reasoning, and memory consolidation via `Ollama` (e.g., LLaMA-3).
*   **Deep Learning Audio Models:** Handles offline Speech-to-Text (`faster-whisper`) and hyper-realistic Text-to-Speech (`Kokoro-82M`).

This hybrid approach ensures that heavy LLM tasks do not block the fluid 60FPS 3D rendering loop, while the lightweight Q-Learning model operates mathematically within the main application thread.

---

## 2. Autonomous Action via Online Reinforcement Learning

Unlike traditional AI models that are pre-trained on massive static datasets, Byte learns *after* you install him. He uses **Continuous Online Reinforcement Learning** to adapt to your specific daily habits.

### The Q-Learning Loop

The core of Byte's decision-making is located in `ReinforcementLearningModel.swift`. It utilizes a mathematically lightweight Q-Table (a matrix of States and Actions).

1.  **State Observation:** Byte constantly observes your environment (`DesktopEnvironmentManager`). The state vector includes:
    *   **Time of Day** (e.g., Morning, Night)
    *   **User Attention** (e.g., Idle, Working, Media Playing)
    *   **Active Windows** (e.g., Code editor open vs. empty desktop)

2.  **Action Selection (Epsilon-Greedy):** When choosing what to do next (Wander, Sleep, Sit, Play), Byte usually picks the action with the highest known "Reward" for the current state (Exploitation). However, 20% of the time (`epsilon = 0.2`), he will pick a completely random action to discover new interactions (Exploration).

3.  **Applying Rewards:** When you interact with Byte (e.g., petting him when he sits next to your code editor, or scolding him for jumping around while you work), the system generates a numerical Reward or Penalty. 

4.  **The Bellman Equation:** The Q-Table mathematically updates instantly based on this reward using the classic Q-Learning formula:
    ```swift
    // Q(s,a) = Q(s,a) + alpha * (reward - Q(s,a))
    let newQ = currentQ + learningRate * (reward - currentQ)
    ```

### Why this is a good approach:
*   **Zero CPU/GPU Bottleneck:** Updating a Double in a Swift dictionary takes nanoseconds. Byte's behavior evolves natively within the app without spinning up heavy Python models or draining your MacBook battery.
*   **Hyper-Personalization:** The model starts blank and shapes itself exactly to *your* routine. If you work late at night, Byte learns that sitting quietly at night yields high rewards, while making noise yields penalties.

## 3. Cognitive Reasoning & Memory Consolidation

While the RL model handles physical habits, cognitive reasoning is offloaded to a local LLM via `AIEngine.swift` and `OllamaClient.swift`.

### Intent Deduction (Zero-Shot)
Instead of rigidly parsing commands like `"set timer"`, Byte passes your transcribed speech to the LLM with the current environment context. The LLM acts as an orchestrator, deducing your *intent* from natural, unstructured dialogue and picking the correct system tool to run.

### The Reflection Engine (Nightly Training)
True intelligence requires long-term memory. Byte utilizes a unique "offline training" loop:

*   During the day, interactions are stored in a short-term memory buffer (`DialogueContextTracker`).
*   When Byte decides to enter a **Sleep Cycle**, he triggers the `ReflectionEngine`.
*   The local LLM is tasked with reading the short-term buffer, analyzing the outcomes, and deducing permanent personality traits or rules.
*   These rules are saved to the `MemoryGraph` and heavily influence the prompt engineering for the next day, ensuring Byte legitimately "remembers" your preferences over time.

---

## 4. Fully Local Voice I/O

To achieve natural interaction without compromising privacy, the voice pipeline is completely decoupled from cloud APIs:

1.  **Hearing:** `VoiceInputManager` captures audio and pipes it to a local `faster-whisper` Python server, running highly optimized STT inference.
2.  **Speaking:** Responses generated by the LLM are sent to `tts_server.py`, where `Kokoro-82M` (a state-of-the-art offline TTS model running on PyTorch) synthesizes natural, emotion-aware audio.

## 5. Summary of Results

By combining **State-Machine architecture, Q-Learning, and Local LLMs**, the following results are achieved:

1.  **Elimination of Robotic Behavior:** Hardcoded pets quickly become predictable and boring. Because Byte balances the Bellman Equation with Epsilon-Greedy exploration and LLM-driven dialogue, his actions are constantly evolving and genuinely surprising.
2.  **Privacy:** Not a single byte of your desktop activity, voice recordings, or application data leaves your machine. 
3.  **Adaptive Routines:** Over time, users find that Byte naturally synchronizes with their workflows—quietly observing during heavy coding sessions, and actively soliciting attention when the user steps away or closes their windows.
